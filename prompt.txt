Fix two related bugs caused by MediaLibrary.createAssetAsync triggering a permission dialog mid-recording. Bug 1 the permissions popup: the media write permission dialog appears because createAssetAsync is called before the user has granted MANAGE_MEDIA or media write permissions. Fix this by requesting all required MediaLibrary permissions at app startup in _layout.tsx before any recording happens, and storing the permission result. Only call createAssetAsync if permission was already granted. If not granted fall back to the documents directory silently without showing any dialog. Bug 2 audio focus loss: the permission dialog steals foreground focus causing expo-av AVManager to call abandonAudioFocus which silences the recording. This is confirmed in adb logs. The fix for Bug 1 should prevent the dialog from appearing during recording entirely. Additionally add an AudioFocus re-request in the AppState active handler so that if audio focus is ever lost during recording the app immediately re-requests it. Run all tests and confirm they pass.
