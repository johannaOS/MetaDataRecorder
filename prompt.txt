Migrate from expo-av to expo-audio for all recording and playback functionality. This is a significant migration that must be done carefully. Here is the complete plan: STEP 1 - Install: run npx expo install expo-audio. STEP 2 - app.json: replace the expo-av plugin entry with the expo-audio plugin configured as follows: microphonePermission Allow VoiceRecorder to record audio, enableBackgroundRecording true, enableBackgroundPlayback true. Remove the withForegroundMicrophoneService plugin and any expo-av or expo-task-manager plugin entries. STEP 3 - Recording in app/(tabs)/index.tsx: replace all expo-av Audio imports with expo-audio imports. Replace Audio.Recording with useAudioRecorder(RecordingPresets.HIGH_QUALITY). Replace Audio.setAudioModeAsync with setAudioModeAsync from expo-audio with allowsRecording true, allowsBackgroundRecording true, playsInSilentMode true, interruptionMode doNotMix. Replace prepareToRecordAsync and startAsync with recorder.prepareToRecordAsync then recorder.record(). Replace stopAndUnloadAsync with recorder.stop() which returns a promise. Get the file URI from recorder.uri after stop. Use useAudioRecorderState(recorder) for durationMillis and isRecording state instead of polling getStatusAsync. Remove all expo-av Audio.Sound playback code from this screen. Remove the backgroundRecording.ts foreground service code since expo-audio handles this natively. STEP 4 - Playback in app/library.tsx: replace expo-av Sound with useAudioPlayer from expo-audio. Replace sound.loadAsync with useAudioPlayer(uri). Replace sound.playAsync with player.play(). Replace sound.pauseAsync with player.pause(). Replace sound.stopAsync and unloadAsync with player.pause() then player.seekTo(0). Use useAudioPlayerStatus(player) for position and duration instead of setOnPlaybackStatusUpdate. STEP 5 - Playback in app/detail/[id].tsx: same pattern as library.tsx. Replace all expo-av Sound usage with useAudioPlayer. Use useAudioPlayerStatus for scrub bar position and duration. Replace setPositionAsync with player.seekTo(seconds). For playback speed use player.setPlaybackRate(rate). STEP 6 - Remove expo-av: remove expo-av from package.json dependencies after migration is complete. STEP 7 - Remove old files: delete lib/backgroundRecording.ts and plugins/withForegroundMicrophoneService.js since expo-audio handles the foreground service natively via the config plugin. STEP 8 - Update all tests to reflect the new API. The key behaviour requirements are: recording must continue when the screen is locked, recording must continue when switching to another app, the timer must show real elapsed time using recorder.currentTime from useAudioRecorderState, saved duration must come from recorder.uri file metadata not from JS timer. Run all tests and confirm they pass. Note that a new native build is required for the background recording config plugin changes to take effect.
