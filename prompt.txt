Fix background recording regression in app/(tabs)/index.tsx. The setAudioModeAsync call was incorrectly made iOS-only in the last change. According to the official expo-audio documentation, setAudioModeAsync with allowsBackgroundRecording true must be called at runtime on both Android and iOS before recording starts - the config plugin alone is not sufficient. Restore setAudioModeAsync to run on both platforms with these settings: playsInSilentMode true, allowsRecording true, allowsBackgroundRecording true. Remove the interruptionMode doNotMix setting entirely on Android as this is known to cause a cast error on Android with expo-audio. Keep interruptionMode doNotMix on iOS only. Run all tests and confirm they pass.
