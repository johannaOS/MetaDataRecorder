Implement background recording and screen lock recording. The app must continue recording without interruption when the user switches to another app or locks the screen. On Android this requires a foreground service with a persistent notification showing recording is in progress with a stop button. Use expo-task-manager and expo-notifications to implement this. The notification should show the app name, a recording indicator, the current duration timer updating every second, and a Stop button that stops the recording cleanly. When the user returns to the app the UI should reflect the current recording state correctly including the elapsed time. The recording must survive screen lock, switching to another app such as a metronome, and the screen turning off to save battery. Add the required permissions to app.json. This requires a new build to take effect - note this clearly in the summary. Run all tests and confirm they pass.
