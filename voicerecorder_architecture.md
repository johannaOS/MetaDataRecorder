# VoiceRecorder — App Architecture

## Overview

Expo managed workflow, React Native + TypeScript. Single Android target (Samsung A55). No bare workflow, no ejection. All native functionality via Expo modules.

---


## Navigation

Expo Router with file-based routing.

- `app/(tabs)/index.tsx` and `app/library.tsx` are root tab screens — back button exits the app
- `app/metadata.tsx`, `app/detail/[id].tsx`, `app/fields.tsx`, `app/settings.tsx` are pushed onto the stack
- After saving metadata, navigation goes to `/library` via `router.replace`
- Library FAB (floating record button) navigates to recorder via `router.replace('/')`

---

## Screen 1 — Recorder (`app/(tabs)/index.tsx`)

**State machine:** `idle` → `recording` → `paused` → back to `recording` or `stopped`

**Audio:** expo-audio
- `useAudioRecorder(RecordingPresets.HIGH_QUALITY)` — recorder hook, always mounted
- `useAudioRecorderState(recorder, 80)` — polls every 80ms for `durationMillis` and `metering`
- `recorder.prepareToRecordAsync()` + `recorder.record()` to start
- `recorder.pause()` / `recorder.record()` for pause/resume
- `recorder.stop()` + `recorder.uri` to stop and get file path

**Before recording starts:**
```
setAudioModeAsync({
  playsInSilentMode: true,
  allowsRecording: true,
  allowsBackgroundRecording: true,
  ...(Platform.OS === 'ios' ? { interruptionMode: 'doNotMix' } : {})
})
```
This call is required on BOTH platforms. The config plugin handles manifest/permissions; this call activates the runtime audio session.

**Timer:** derived reactively — `Math.round((recorderState.durationMillis ?? 0) / 1000)`

**Waveform:** driven by `recorderState.metering` via useEffect watching `recorderState.durationMillis`

**Inline metadata form:** split-screen while recording — user can fill fields before stopping

**Back button:** shows discard confirmation if recording is active

## Screen 2 — Metadata Form (`app/metadata.tsx`)

- Full metadata entry: title, of/after (max 2 words), from (max 1 word), song type, who's playing, notes
- Autofill via `lib/autoFill.ts`
- On save: writes to SQLite, copies audio file to final destination via `lib/saveRecording.ts`, navigates to `/library`
- Field visibility controlled by `useFieldConfig` hook

---

## Screen 3 — Library (`app/library.tsx`)

- Lists recordings newest-first from SQLite
- Search bar with X clear button
- Dynamic filter keywords
- Long-press → edit or delete
- Floating record button (FAB)
- Inline playback via `createAudioPlayer(source)` (imperative, ref-managed)
- Playback stops on focus loss via `useFocusEffect` cleanup: `player.pause()` then `player.remove()`

---

## Screen 4 — Playback/Detail (`app/detail/[id].tsx`)

- `useAudioPlayer(uri)` + `useAudioPlayerStatus(player)`
- Scrub bar: local `seekPositionMs` state during drag, `player.seekTo(seconds)` on release
- Playback speed: `player.setPlaybackRate(rate)`
- Position from `playerStatus.currentTime * 1000`, duration from `playerStatus.duration * 1000`
- Inline edit of metadata
- Delete with confirmation

## File Storage (`lib/saveRecording.ts`)

Two-tier strategy, no MediaLibrary (removed — caused duplicate files and permission popups):

**Tier 1:** Direct write to `/storage/emulated/0/Music/VoiceRecorder/`
- Android 9/10 direct filesystem access
- Filename generated by `lib/filename.ts` — safe chars, Swedish chars preserved, conflict resolution with (1)(2)...

**Tier 2:** App documents directory fallback
- Used when Tier 1 fails (permissions, Android 11+)

The expo-audio recorder saves to a temp URI in app cache (`file:///data/user/0/com.johannaos.VoiceRecorder/cache/Audio/recording-xxx.m4a`). saveRecording.ts copies this to the final destination.

## Field Management (`hooks/useFieldConfig.ts`)

- Shared hook used by Screen 1 (inline form), Screen 2 (full form), Screen 4 (inline edit)
- Returns `[configs, reloadFn]`
- Max 20 fields total (counts hidden fields)
- Changes take effect instantly via `useFocusEffect`

---

## Startup (`app/_layout.tsx`)

Sequential permissions on startup (never simultaneous — causes Android deadlock):
1. Microphone permission
2. Notifications permission
3. `initRecordingNotifications()`
4. `autoBackupOnStartup()` — backs up SQLite to documents/backups/, keeps 3 most recent

---

## Background Recording

expo-audio config plugin handles:
- `AndroidManifest.xml` foreground service declaration with `foregroundServiceType="microphone"`
- iOS `UIBackgroundModes: audio`
- `FOREGROUND_SERVICE_MICROPHONE` permission

Runtime `setAudioModeAsync` with `allowsBackgroundRecording: true` is additionally required — the config plugin alone is not sufficient.

The foreground service shows a persistent "Recording audio" notification while recording. This cannot be dismissed while recording is active.

---

## Backup (`lib/backup.ts`)

- Auto-backup on startup to `documents/backups/`, keeps 3 most recent
- Manual backup/restore via Settings screen (gear icon in Library header)

## Key Rules (from CLAUDE.md)

- Always use safe area insets pattern
- Every change requires 125 passing tests
- Sequential permissions at startup, never simultaneous
- `setAudioModeAsync` on both platforms, `interruptionMode` iOS only
- No MediaLibrary
- Always websearch current docs before touching native audio APIs
